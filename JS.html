async和defer

  1. <script src="script.js" async><"/script>async告知浏览器在遇到 <script> 元素时不要中断后续 HTML 内容的加载。因此无需使用 DOMContentLoaded 事件
  2. 浏览器遇到 async 脚本时不会阻塞页面渲染，而是直接下载然后运行。这样脚本的运行次序就无法控制，只是脚本不会阻止剩余页面的显示。
  3. 如果脚本无需等待页面解析，且无依赖独立运行，那么应使用 async。
  4. 如果脚本需要等待解析，且依赖于其它脚本，调用这些脚本时应使用 defer，将关联的脚本按所需顺序置于 HTML 中。
